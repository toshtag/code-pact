# Cycle Correlation Propagation

**Status:** accepted (P72-T3, 2026-07)
**Scope:** define complete task-loop coverage and automatic cycle-ref
propagation before P66 runtime work begins.
**Owners:** maintainer
**Related:** P71-T1, P71-T2, P72-T1, P72-T2

## Cycle Ref Surfaces

P66 adds optional `--cycle-ref <ref>` to:

```text
task start
task resume
task block
task context
verify
task complete
task record-done
context show
evidence show
memory recall
```

P66 does not add `--cycle-ref` to `task finalize`.

## Generated Commands

When tracked `task prepare` succeeds and starts or resumes a cycle, returned
commands include the same cycle ref:

```text
commands.start
commands.context
commands.verify
commands.complete
commands.record-done
```

Default, untracked `task prepare` output remains byte-identical.

Example tracked verify command:

```text
code-pact verify --phase P1 --task P1-T1 --json --detail agent --cycle-ref cycle:sha256:...
```

## Deferred Context

If a tracked cycle returns deferred context commands, each
`deferred_context.retrieve_command` includes the same cycle ref.

## Evidence Retrieval

If tracked `task complete` returns an Evidence retrieval command after a
failure, that command includes the same cycle ref:

```text
code-pact evidence show evidence:sha256:... --cycle-ref cycle:sha256:... --json
```

## Memory Retrieval

P65 does not expose a cycle flag. After P66 adds metrics runtime, any generated
memory recall command that can be associated with the current cycle includes
the cycle ref explicitly.

## Verify Coverage

P66 records standalone verify stdout with categories:

```text
verify_success
verify_failure
```

`verification_run_count` counts both:

- standalone `verify`
- verification performed inside `task complete`

Running the same verification twice counts twice.

## Task Control Coverage

Small lifecycle command stdout is counted under:

```text
task_control
```

This includes:

```text
task start
task resume
task block
task record-done
```

`task finalize` remains excluded because it is post-cycle governance.

## Unknown Ref

For unknown, pruned, or other-repository refs:

- the command's main result does not change
- correctness exit code does not change
- stdout content does not change
- JSON envelope commands may add an additive warning
- raw context and Evidence output use stderr warnings only
- no command assigns output to the latest open cycle by inference

## Consequences

The standard task loop is trackable without requiring the Agent to manually
edit every returned command. Missing cycle refs are no longer an expected
failure mode for commands generated by tracked `task prepare`.
